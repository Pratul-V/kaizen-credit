# 🏦 Kaizen Credit — AI-Powered Credit Risk Intelligence System

**Kaizen Credit** is an advanced, enterprise-inspired, and fully dynamic Credit Risk Intelligence platform built using Python, Flask, and XGBoost. Designed to simulate modern fintech underwriting systems, the platform goes beyond traditional credit scoring by integrating **Explainable AI (XAI)**, intelligent **risk-based pricing**, automated **counter-offer generation**, and interactive **financial simulation tools** into a unified ecosystem.

The name **“Kaizen”** originates from the Japanese philosophy of **continuous improvement**. In the context of this platform, the name represents:

* intelligent financial optimization
* adaptive risk analysis
* data-driven decision refinement
* continuously improving underwriting decisions

Combined with the word **“Credit”**, the brand identity reflects a modern AI-powered financial platform focused on smarter, more transparent, and more reliable lending intelligence.

> **Kaizen Credit**
> *Smarter Credit, Stronger Decisions.*

Unlike conventional machine learning dashboards, this system focuses heavily on:

* AI transparency
* business-rule enforcement
* underwriting realism
* financial decision intelligence
* professional UI/UX architecture

The platform is completely database-free and operates dynamically using uploaded datasets and in-memory processing, making it lightweight, deployment-friendly, and highly modular.

This project demonstrates the practical intersection of:

* Machine Learning
* Explainable AI
* Financial Risk Analytics
* Business Logic Engineering
* FinTech Product Design
* Interactive Data Visualization

# ✨ Core Features

## 🧠 Explainable AI (XAI) with SHAP

The platform integrates **SHAP (SHapley Additive exPlanations)** alongside XGBoost to provide transparent and interpretable AI decisions.

Instead of producing black-box predictions, the system explains:

* why an applicant was classified as low or high risk
* which financial variables influenced the decision
* how strongly each feature contributed to the prediction

The explainability engine generates:

* feature importance analysis
* SHAP-based contribution breakdowns
* human-readable underwriting explanations
* dynamic visual analytics

This makes the AI system significantly more transparent, analyst-friendly, and aligned with modern financial AI standards.

---

## 🛡️ Business Rules Guardrails

The system incorporates a two-tier financial validation framework to ensure all AI outputs remain aligned with realistic underwriting policies.

### Hard Rejections

Applications are automatically blocked if they violate strict business constraints such as:

* invalid age ranges
* impossible employment history
* negative income values
* extreme debt-to-income ratios

### Soft Warnings

Potentially risky applications are flagged for review without fully blocking predictions.

Examples include:

* unusually high loan requests
* elevated liabilities
* inconsistent financial behaviour

This hybrid AI + business-rule architecture prevents unrealistic approvals and improves decision reliability.

---

## 🧮 Dynamic Pricing & Intelligent Counter-Offers

One of the platform’s most advanced features is its intelligent risk-based pricing engine.

Instead of simply rejecting high-risk applicants, the system dynamically recalculates:

* revised loan amounts
* adjusted interest rates
* optimized repayment structures
* affordability-based EMI plans

The platform then generates automated counter-offers tailored to the applicant’s financial profile.

Example:

* Requested Loan: ₹15,00,000
* High Risk Prediction
* Counter-Offer:

  * Approved Amount: ₹8,00,000
  * Revised Interest Rate
  * Optimized EMI Structure

This closely mirrors real-world fintech underwriting workflows used by modern lending institutions.

---

# 📂 Project Structure

```text id="r3wd82"
├── app.py                      # Main Flask application entry point
├── config.py                   # Centralized configuration (Risk tiers, Business rules)
├── train_model.py              # Script to train the XGBoost model and save artifacts
├── data/                       # Directory for training datasets
├── models/                     # Pickled XGBoost model and SHAP explainer
├── modules/
│   ├── model_loader.py         # Lazy singleton loader for AI models
│   ├── pdf_generator.py        # PDF report generation logic
│   ├── predictor.py            # AI prediction and SHAP value mapping
│   ├── validator.py            # Business rule validation engine
│   └── pricing/                # Dynamic pricing and counter-offer algorithms
├── routes/                     # Flask Blueprint controllers
│   ├── dashboard.py            # System Intelligence Dashboard route
│   ├── predict.py              # Main risk analysis and PDF download route
│   ├── shap_api.py             # JSON API for SHAP explainability charts
│   └── simulate.py             # What-If Simulator routes
├── static/                     # Vanilla CSS design system and assets
└── templates/                  # HTML templates (Jinja2)
```

## Clone the Repository

```bash

cd kaizen-credit
```

## Create Virtual Environment

```bash
python -m venv venv
```

#### Windows

```bash
venv\Scripts\activate
```

#### macOS/Linux

```bash
source venv/bin/activate
```

---

```bash
pip install -r requirements.txt
```

---

## Run the Application

```bash
python app.py
```

The application will start locally at:

```text

```
